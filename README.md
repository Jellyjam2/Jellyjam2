## 🛡️ Engineering Direction

I build systems around a few simple rules:

```text
inspect → branch → patch → test → commit → PR → merge → release
```

Core principles:

- security-first design
- local-first workflows where possible
- reproducible setup before public claims
- clear logs before big promises
- no hidden system changes
- no secret keys in repositories
- documentation that explains what is real, what is experimental, and what is next
